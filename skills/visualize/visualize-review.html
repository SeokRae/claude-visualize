Issue #1
모바일 브레이크포인트가 디자인 규칙을 위반
버그

디자인 규칙은 "Body는 15px 또는 16px — 설명 텍스트에 절대 14px을 기본값으로 사용하지 말 것"이라고 명시하지만, 스켈레톤의 모바일 미디어 쿼리는 p { font-size: 14px }를 직접 적용하여 규칙과 정면 충돌합니다.

위치 HTML Skeleton → @media (max-width: 375px) 블록
규칙 출처 Design Rules → Typography → Body Text
수정 방법 font-size: 15px로 변경하거나, 해당 오버라이드 자체를 제거
Issue #2
.grid.four CSS 클래스 누락
버그

디자인 규칙에 "4-col은 짧은 KPI, 상태 pill, 숫자 카드 전용"이라는 명세가 있지만, 스켈레톤은 .grid.two와 .grid.three만 정의하고 .grid.four는 구현되지 않았습니다. 이 클래스를 사용하는 AI 생성 코드는 레이아웃이 전혀 적용되지 않는 상태로 출력됩니다.

위치 HTML Skeleton → .grid 정의 블록
수정 방법 아래 CSS 추가 + 모바일에서 2-col로 collapse 
/* 추가할 코드 */
.grid.four {
  grid-template-columns: repeat(4, minmax(0, 1fr));
}

@media (max-width: 768px) {
  .grid.four {
    grid-template-columns: repeat(2, minmax(0, 1fr));
  }
}
